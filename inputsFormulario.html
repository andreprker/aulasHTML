Inputs Variados em Formulários HTML

O elemento <input> é um dos mais versáteis em HTML para a criação de formulários. Ele permite que os usuários interajam com a página, fornecendo dados de diversas formas. A principal característica que define o tipo de interação e o formato de dados esperados é o atributo type. Vamos explorar os tipos mais comuns de <input>.

---

Tipos Comuns de <input>

type="text": Campo de Texto Simples

Usado para entrada de texto de uma única linha. É o tipo de input mais básico e comum.

<label for="nome">Nome:</label>
<input type="text" id="nome" name="nome_usuario">

type="password": Campo de Senha

Similar ao text, mas os caracteres digitados são ocultados (geralmente com asteriscos ou pontos) para privacidade.

<label for="senha">Senha:</label>
<input type="password" id="senha" name="senha_usuario">

type="email": Campo de Email

Projetado para endereços de email. Navegadores modernos podem fornecer validação automática básica do formato do email e teclados otimizados em dispositivos móveis.

<label for="email">Email:</label>
<input type="email" id="email" name="email_usuario">

type="number": Campo Numérico

Usado para entrada de números. Navegadores podem exibir controles de incremento/decremento (spinners) e teclados numéricos em dispositivos móveis. Permite atributos como min, max e step.

<label for="quantidade">Quantidade:</label>
<input type="number" id="quantidade" name="qtd" min="1" max="10" step="1">

type="date": Campo de Data

Permite a seleção de uma data. Navegadores geralmente fornecem um seletor de calendário (date picker) para facilitar a entrada.

<label for="data_nascimento">Data de Nascimento:</label>
<input type="date" id="data_nascimento" name="data_nasc">

type="checkbox": Caixa de Seleção

Usado para permitir que o usuário selecione uma ou mais opções de um conjunto. Cada checkbox é um item independente.

<input type="checkbox" id="aceitar_termos" name="termos" value="sim">
<label for="aceitar_termos">Aceito os termos e condições</label><br>

<input type="checkbox" id="receber_noticias" name="noticias" value="sim" checked>
<label for="receber_noticias">Desejo receber notícias</label>

type="radio": Botão de Rádio

Usado para permitir que o usuário selecione apenas uma opção de um grupo. Vários botões de rádio no mesmo grupo devem ter o mesmo atributo name.

<p>Selecione seu gênero:</p>
<input type="radio" id="genero_masc" name="genero" value="masculino">
<label for="genero_masc">Masculino</label><br>
<input type="radio" id="genero_fem" name="genero" value="feminino">
<label for="genero_fem">Feminino</label>

Selecione seu gênero:

type="submit": Botão de Envio

Cria um botão que, quando clicado, envia os dados do formulário para o URL especificado no atributo action do <form>.

<button type="submit">Enviar Formulário</button>

type="reset": Botão de Reset

Cria um botão que, quando clicado, redefine todos os campos do formulário para seus valores iniciais.

<button type="reset">Limpar Formulário</button>

Exemplo Completo com Diversos Tipos de Input

Este formulário demonstra a utilização de vários tipos de <input> em conjunto:

Código HTML:

<h2>Formulário de Inscrição</h2>
<form action="/processar_inscricao.php" method="POST">
    <label for="nome_completo">Nome Completo:</label>
    <input type="text" id="nome_completo" name="nome" required>

    <label for="email_inscricao">Email:</label>
    <input type="email" id="email_inscricao" name="email" required>

    <label for="senha_nova">Criar Senha:</label>
    <input type="password" id="senha_nova" name="senha" required>

    <label for="idade_usuario">Idade:</label>
    <input type="number" id="idade_usuario" name="idade" min="18" max="99" required>

    <label for="data_evento">Data Preferida para Evento:</label>
    <input type="date" id="data_evento" name="data_evento">

    <p><strong>Interesses:</strong></p>
    <input type="checkbox" id="dev_web" name="interesses" value="web">
    <label for="dev_web">Desenvolvimento Web</label><br>
    <input type="checkbox" id="dev_mobile" name="interesses" value="mobile">
    <label for="dev_mobile">Desenvolvimento Mobile</label><br>
    <input type="checkbox" id="ia" name="interesses" value="ia">
    <label for="ia">Inteligência Artificial</label><br>

    <p><strong>Nível de Experiência:</strong></p>
    <div class="radio-group">
        <input type="radio" id="junior" name="nivel_experiencia" value="junior">
        <label for="junior">Júnior</label>
        <input type="radio" id="pleno" name="nivel_experiencia" value="pleno">
        <label for="pleno">Pleno</label>
        <input type="radio" id="senior" name="nivel_experiencia" value="senior">
        <label for="senior">Sênior</label>
    </div>

    <label for="comentarios">Comentários Adicionais:</label>
    <textarea id="comentarios" name="comentarios" rows="4"></textarea>

    <button type="submit">Inscrever-se</button>
    <button type="reset">Limpar</button>
</form>

Conclusão

A diversidade de tipos de <input> oferece flexibilidade enorme para criar formulários que se adaptam a praticamente qualquer necessidade de coleta de dados. Usar o tipo correto de <input> não só melhora a experiência do usuário (UX) com validações e interfaces otimizadas, mas também contribui para a semântica e acessibilidade da sua página web.
