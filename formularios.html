Elemento <form> com Atributos action e method

O elemento HTML <form> é essencial para coletar informações do usuário e enviá-las para um servidor web. Ele serve como um contêiner para diversos controles de entrada, como campos de texto, caixas de seleção, botões de rádio e botões de envio. Para que os dados coletados sejam processados, o formulário utiliza dois atributos principais: action e method.

---

Atributo action

O atributo action especifica a URL para onde os dados do formulário serão enviados quando ele for submetido. Esta URL geralmente aponta para um script ou programa no servidor (por exemplo, um arquivo PHP, Python, Node.js, etc.) que será responsável por processar as informações.

Se o atributo action for omitido, os dados serão enviados para a URL da página atual.

Sintaxe:

<form action="url_para_processamento">
    <!-- Campos do formulário -->
</form>

Exemplo:

<form action="/processar_cadastro.php">
    <!-- ... -->
</form>

No exemplo, quando o formulário for submetido, os dados serão enviados para o script processar_cadastro.php localizado na raiz do servidor.

---

Atributo method

O atributo method especifica o método HTTP a ser usado ao enviar os dados do formulário. Os dois valores mais comuns para este atributo são GET e POST.

Método GET

  • Os dados do formulário são anexados à URL como pares nome/valor (query string) e enviados.
  • Os dados são visíveis na barra de endereço do navegador.
  • Existe um limite no tamanho dos dados que podem ser enviados.
  • É adequado para envios de dados que não alteram o estado do servidor (ex: buscas, filtragens).
  • Os dados podem ser marcados como favoritos e compartilhados.

Sintaxe e Exemplo:

<form action="/buscar.php" method="GET">
    <label for="termo">Termo de Busca:</label>
    <input type="text" id="termo" name="q">
    <button type="submit">Buscar</button>
</form>

Ao submeter o formulário acima com "HTML" no campo de busca, a URL ficaria algo como: /buscar.php?q=HTML.

Método POST

  • Os dados do formulário são enviados no corpo da requisição HTTP.
  • Os dados não são visíveis na URL, tornando-o mais seguro para informações sensíveis (senhas, dados pessoais).
  • Não há um limite prático para o tamanho dos dados que podem ser enviados.
  • É adequado para envios de dados que alteram o estado do servidor (ex: cadastro, login, envio de arquivos).
  • Não é possível marcar a URL como favorita com os dados do formulário.

Sintaxe e Exemplo:

<form action="/salvar_usuario.php" method="POST">
    <label for="nome">Nome:</label>
    <input type="text" id="nome" name="nome_usuario">
    <label for="email">Email:</label>
    <input type="email" id="email" name="email_usuario">
    <button type="submit">Cadastrar</button>
</form>

Ao submeter o formulário acima, os dados serão enviados no corpo da requisição para /salvar_usuario.php, sem serem visíveis na URL.

---

Exemplo Completo de Formulário com action e method

Aqui está um formulário de contato simples que demonstra o uso de ambos os atributos:

Código HTML:

<h2>Formulário de Contato</h2>
<form action="/enviar_mensagem.php" method="POST">
    <label for="nome_completo">Nome Completo:</label>
    <input type="text" id="nome_completo" name="nome" required>

    <label for="email_contato">Seu Email:</label>
    <input type="email" id="email_contato" name="email" required>

    <label for="mensagem_contato">Mensagem:</label>
    <textarea id="mensagem_contato" name="mensagem" rows="5" required></textarea>

    <button type="submit">Enviar Mensagem</button>
</form>

---

Conclusão

Os atributos action e method do elemento <form> são cruciais para definir como e para onde os dados do usuário serão enviados. A escolha entre os métodos GET e POST depende da natureza dos dados e da operação que será realizada no servidor. Compreender o funcionamento desses atributos é fundamental para criar formulários web eficientes e seguros.
